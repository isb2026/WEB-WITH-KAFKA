# 📦 SCM 프로젝트 문서

## 📋 **프로젝트 개요**

SCM(Supply Chain Management)은 공급업체부터 고객까지의 전체 공급망을 효율적으로 관리하고 최적화하는 현대적인 웹 애플리케이션입니다.

### **🏗️ 기술 스택**

- **Frontend**: React 18.3.1 + TypeScript 5.7.2
- **UI Framework**: Radix UI + Tailwind CSS
- **State Management**: React Query + React Table
- **Charts**: ECharts (공급망 시각화 최적화)
- **Build Tool**: Vite 6.2.0
- **Package Manager**: pnpm

### **📊 현재 상태**

- **완성도**: 🟠 10%
- **개발 단계**: 초기 개발 단계
- **주요 기능**: 공급업체 관리, 재고 관리, 물류 관리
- **특화 영역**: 공급망 최적화 및 리스크 관리

## 🎯 **SCM 핵심 기능**

### **🏢 공급업체 관리 (Supplier Management)**

#### **공급업체 등록 및 정보 관리**

- **기본 정보**: 업체명, 사업자등록번호, 연락처, 주소
- **업무 정보**: 주요 제품/서비스, 생산능력, 품질 인증
- **재무 정보**: 매출액, 신용등급, 재무상태
- **계약 정보**: 계약 조건, 납기, 품질 기준

#### **공급업체 평가 및 성과 관리**

- **품질 평가**: 제품 품질, 납기 준수율, 불량률
- **비용 평가**: 가격 경쟁력, 총소유비용(TCO)
- **서비스 평가**: 응답성, 유연성, 기술 지원
- **리스크 평가**: 재무 안정성, 운영 연속성

#### **공급업체 관계 관리**

- **계약 관리**: 계약 조건, 갱신, 변경 관리
- **성과 개선**: 정기 평가 및 개선 계획 수립
- **협력 프로그램**: 공동 개발, 품질 향상 협력
- **이탈 관리**: 공급업체 이탈 위험 및 대응 방안

### **📦 재고 관리 (Inventory Management)**

#### **재고 수준 관리**

- **현재 재고**: 실시간 재고 수량 및 위치 추적
- **안전재고**: 최소 재고 수준 설정 및 관리
- **최대 재고**: 과잉 재고 방지를 위한 상한선
- **재고 회전율**: 재고 회전 속도 및 효율성 분석

#### **주문점 및 재주문 관리**

- **주문점 설정**: 자동 재주문 트리거 수준
- **경제적 주문량**: 총비용 최소화를 위한 주문량 계산
- **재주문 주기**: 공급업체별 납기 및 주문 주기
- **긴급 주문**: 예상치 못한 수요 증가 시 긴급 조달

#### **재고 최적화**

- **ABC 분석**: 재고 가치별 분류 및 관리 전략
- **수명주기 관리**: 제품 수명주기별 재고 전략
- **크로스 도킹**: 입고와 출고를 동시에 처리
- **재고 예측**: 수요 예측 기반 재고 계획

### **🚚 물류 관리 (Logistics Management)**

#### **운송 관리**

- **운송 계획**: 최적 경로 및 운송 수단 선택
- **운송 추적**: 실시간 위치 추적 및 상태 모니터링
- **운송비 관리**: 운송 비용 최적화 및 분석
- **운송사 관리**: 운송사 평가 및 성과 관리

#### **배송 관리**

- **배송 계획**: 배송 일정 및 경로 최적화
- **배송 추적**: 고객별 배송 상태 추적
- **배송 증명**: 배송 완료 확인 및 서명 수집
- **반품 관리**: 반품 처리 및 원인 분석

#### **창고 관리**

- **창고 레이아웃**: 효율적인 공간 활용 및 배치
- **입출고 관리**: 입고, 보관, 출고 프로세스
- **피킹 최적화**: 주문별 효율적인 상품 집품
- **창고 자동화**: 자동화 시스템 및 장비 관리

### **📈 수요 계획 (Demand Planning)**

#### **수요 예측**

- **시계열 분석**: 과거 데이터 기반 수요 패턴 분석
- **계절성 분석**: 계절별 수요 변동 패턴
- **트렌드 분석**: 장기 수요 변화 추세
- **외부 요인**: 시장 상황, 경쟁사 동향 반영

#### **수요 계획 수립**

- **단기 계획**: 월별, 분기별 수요 계획
- **중기 계획**: 연간 수요 및 생산 계획
- **장기 계획**: 전략적 수요 및 역량 계획
- **시나리오 계획**: 다양한 상황별 대응 계획

#### **수요 실행 관리**

- **계획 실행**: 수요 계획의 실제 실행 및 모니터링
- **계획 조정**: 실제 수요와 계획의 차이 분석 및 조정
- **성과 평가**: 수요 계획 정확도 및 효과성 평가
- **지속 개선**: 예측 모델 및 계획 프로세스 개선

### **⚠️ 리스크 관리 (Risk Management)**

#### **공급망 리스크 식별**

- **공급업체 리스크**: 공급업체 이탈, 품질 문제, 납기 지연
- **수요 리스크**: 수요 급증, 수요 감소, 수요 변동성
- **운송 리스크**: 운송 지연, 운송사 문제, 자연재해
- **재고 리스크**: 과잉 재고, 재고 부족, 재고 손실

#### **리스크 평가 및 우선순위**

- **발생 확률**: 리스크 발생 가능성 평가
- **영향도**: 리스크 발생 시 비즈니스 영향 평가
- **우선순위**: 확률과 영향도를 고려한 리스크 등급
- **대응 비용**: 리스크 대응을 위한 비용 및 자원

#### **리스크 대응 전략**

- **리스크 회피**: 리스크 발생 가능성 제거
- **리스크 감소**: 리스크 발생 확률 또는 영향도 감소
- **리스크 이전**: 보험, 계약 등을 통한 리스크 이전
- **리스크 수용**: 일정 수준의 리스크 수용 및 모니터링

## 🚀 **빠른 시작**

### **개발 환경 설정**

```bash
# 프로젝트 루트에서
cd apps/scm

# 의존성 설치
pnpm install

# 개발 서버 시작
pnpm dev
```

### **SCM 컴포넌트 사용**

```typescript
// 공급업체 관리 컴포넌트
import { SupplierManagement } from '@repo/scm/components';

<SupplierManagement
  suppliers={supplierData}
  onSupplierUpdate={handleSupplierUpdate}
  onSupplierDelete={handleSupplierDelete}
/>

// 재고 관리 컴포넌트
import { InventoryDashboard } from '@repo/scm/components';

<InventoryDashboard
  inventory={inventoryData}
  thresholds={thresholdSettings}
  onReorder={handleReorder}
/>
```

### **공급망 시각화**

```typescript
// 공급망 네트워크 맵
import { SupplyChainMap } from '@repo/scm/visualization';

<SupplyChainMap
  nodes={supplierNodes}
  edges={supplyRelations}
  onNodeClick={handleNodeClick}
  onPathAnalysis={handlePathAnalysis}
/>
```
