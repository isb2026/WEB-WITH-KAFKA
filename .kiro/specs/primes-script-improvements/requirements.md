# Primes Script 코드 생성 개선 요구사항

## 소개

Primes 앱의 스크립트 기반 코드 생성 시스템에서 발생하는 코드 품질 및 호환성 문제들을 해결하여 더 안정적이고 자연스러운 코드를 생성하도록 개선합니다.

## 요구사항

### 요구사항 1: Node.js 버전 호환성 개선

**사용자 스토리:** 개발자로서 Node.js 20 환경에서 스크립트가 안정적으로 실행되기를 원합니다.

#### 승인 기준
1. WHEN 스크립트를 실행할 때 THEN Node.js 20 환경에서 문법 오류 없이 실행되어야 함
2. WHEN Optional chaining 연산자(?.)를 사용할 때 THEN 구형 문법으로 변환되어야 함
3. WHEN 스크립트 파일을 실행할 때 THEN nvm use 20 환경 설정이 자동으로 적용되어야 함

### 요구사항 2: 변수명 생성 개선

**사용자 스토리:** 개발자로서 하이픈(-)이 포함된 모듈명이 올바른 camelCase 변수명으로 변환되기를 원합니다.

#### 승인 기준
1. WHEN 모듈명에 하이픈(-)이 포함될 때 THEN camelCase로 변환되어야 함 (예: tax-invoice → taxInvoice)
2. WHEN 훅 이름을 생성할 때 THEN 하이픈이 제거되고 camelCase로 변환되어야 함
3. WHEN 컴포넌트명을 생성할 때 THEN PascalCase로 올바르게 변환되어야 함

### 요구사항 3: TabNavigation useEffect 로직 개선

**사용자 스토리:** 개발자로서 TabNavigation의 경로 매칭 로직이 더 안정적이고 자연스럽게 작동하기를 원합니다.

#### 승인 기준
1. WHEN location.pathname이 변경될 때 THEN 정확한 탭이 활성화되어야 함
2. WHEN 경로 매칭을 수행할 때 THEN 부분 문자열 매칭 대신 정확한 경로 매칭을 사용해야 함
3. WHEN useEffect 의존성 배열을 설정할 때 THEN location.pathname만 포함해야 함

### 요구사항 4: 컬럼 정의 개선

**사용자 스토리:** 개발자로서 빈 컬럼 배열 대신 실제 사용 가능한 기본 컬럼이 생성되기를 원합니다.

#### 승인 기준
1. WHEN 컬럼 정보가 비어있을 때 THEN 기본 컬럼 세트가 생성되어야 함
2. WHEN 날짜 필드가 있을 때 THEN 적절한 cell 렌더러가 추가되어야 함
3. WHEN 컬럼 크기가 지정되지 않았을 때 THEN 기본 크기가 설정되어야 함

### 요구사항 5: 타입 정의 개선

**사용자 스토리:** 개발자로서 생성되는 코드에 적절한 TypeScript 타입 정의가 포함되기를 원합니다.

#### 승인 기준
1. WHEN 컴포넌트를 생성할 때 THEN Props 인터페이스가 정의되어야 함
2. WHEN 데이터 타입을 사용할 때 THEN 적절한 인터페이스가 생성되어야 함
3. WHEN 상태 변수를 선언할 때 THEN 타입이 명시되어야 함

### 요구사항 6: 에러 처리 및 로딩 상태 개선

**사용자 스토리:** 개발자로서 생성되는 코드에 적절한 에러 처리와 로딩 상태가 포함되기를 원합니다.

#### 승인 기준
1. WHEN API 호출을 수행할 때 THEN 로딩 상태가 관리되어야 함
2. WHEN 에러가 발생할 때 THEN 적절한 에러 처리가 포함되어야 함
3. WHEN 폼 제출을 수행할 때 THEN 중복 제출 방지 로직이 포함되어야 함

### 요구사항 7: 코드 품질 개선

**사용자 스토리:** 개발자로서 생성되는 코드가 일관된 스타일과 모범 사례를 따르기를 원합니다.

#### 승인 기준
1. WHEN 코드를 생성할 때 THEN 주석 처리된 코드가 없어야 함
2. WHEN import 문을 생성할 때 THEN 사용되지 않는 import가 없어야 함
3. WHEN 함수를 생성할 때 THEN useCallback 등 성능 최적화가 적절히 적용되어야 함
